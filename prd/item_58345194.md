# 무추망망이 이미지 자동 크롭 시스템 PRD

## Overview

무추망망이 프로젝트의 일환으로 개발되는 지능형 이미지 자동 크롭 시스템. 스마트폰 메시지 화면 스크린샷에서 캐릭터 이미지나 특정 영역을 자동으로 감지하고 정확히 크롭하는 것이 핵심 기능이다. OpenCV 기반의 컴퓨터 비전 기술을 활용하여 사용자의 수동 작업 없이도 원하는 영역을 지능적으로 추출한다.

## Core Value

**스마트폰 메시지 스크린샷에서 캐릭터 이미지를 사람의 개입 없이 정확하게 자동 추출하는 것**

이는 기존의 범용 이미지 크롭 도구와 차별화되는 핵심 가치로, 특정 도메인(모바일 메시지 화면)에 특화된 자동 감지 알고리즘을 통해 사용자가 일일이 영역을 지정할 필요를 제거한다.

## MVP Scope

### 포함되는 기능
- **자동 캐릭터 이미지 감지**: OpenCV를 활용한 모바일 메시지 화면 내 캐릭터 이미지 자동 인식
- **지능형 크롭 실행**: 감지된 영역을 자동으로 크롭하여 별도 이미지로 추출
- **기본적인 사용자 인터페이스**: 이미지 업로드 및 결과 확인이 가능한 최소한의 UI
- **결과 이미지 저장**: 크롭된 이미지를 사용자가 저장할 수 있는 기능

### 의도적으로 제외되는 기능 (v1에서)
- 다중 캐릭터 동시 감지 및 크롭
- 메시지 화면 외 다른 타입 이미지 처리
- 이미지 편집 기능 (회전, 필터, 색상 조정 등)
- 배치 처리 기능
- 클라우드 저장소 연동
- 사용자 계정 및 인증 시스템
- 모바일 앱 버전

## Requirements

### 기능적 요구사항

1. **이미지 입력 처리**
   - 일반적인 스마트폰 스크린샷 해상도 지원 (최소 1080x1920)
   - PNG, JPG 형식 지원
   - 파일 크기 제한: 10MB 이하

2. **자동 감지 알고리즘**
   - OpenCV 기반 객체 감지로 메시지 화면 내 캐릭터 이미지 식별
   - 프로필 사진, 이모지, 스티커 등 시각적으로 구분되는 요소 감지
   - 최소 80% 정확도 목표

3. **크롭 실행**
   - 감지된 영역 주변에 적절한 여백 포함하여 크롭
   - 원본 이미지 품질 유지
   - 크롭 영역이 너무 작거나 큰 경우 필터링

4. **사용자 인터페이스**
   - 드래그 앤 드롭 이미지 업로드
   - 감지 결과 미리보기
   - 크롭된 이미지 다운로드

### 비기능적 요구사항

1. **성능**
   - 이미지 처리 시간: 5초 이내
   - 동시 처리: 최소 10명 사용자 지원

2. **사용성**
   - 기술적 지식 없는 사용자도 3번의 클릭으로 결과 획득
   - 감지 실패 시 명확한 에러 메시지 제공

3. **호환성**
   - 모던 웹 브라우저 지원 (Chrome, Firefox, Safari, Edge 최신 2개 버전)

## Technical Considerations

### 기술 스택 (가정)
- **백엔드**: Python + FastAPI (OpenCV 연동 용이성)
- **프론트엔드**: HTML/CSS/JavaScript (단순한 MVP용)
- **이미지 처리**: OpenCV + NumPy
- **배포**: Docker 컨테이너

### 핵심 기술적 도전과제
1. **메시지 UI 패턴 인식**: 다양한 메신저 앱의 UI 레이아웃 차이 처리
2. **캐릭터 이미지 정의**: 프로필 사진 vs 이모지 vs 스티커 구분 기준
3. **크롭 영역 최적화**: 캐릭터 주변의 적절한 여백 계산

### 아키텍처 원칙
- 이미지 처리는 서버사이드에서 수행 (OpenCV 의존성)
- 스테이트리스 설계: 각 요청 독립적 처리
- 임시 파일 관리: 처리 완료 후 자동 정리

## Risks & Assumptions

### 주요 가정
1. **대상 이미지의 일관성**: 모바일 메시지 스크린샷은 비교적 일관된 UI 패턴을 가진다고 가정
2. **캐릭터 이미지의 시각적 특성**: 배경과 구분되는 명확한 경계선을 가진다고 가정
3. **사용자 요구의 명확성**: "캐릭터 이미지"에 대한 사용자의 기대가 일관적이라고 가정

### 기술적 리스크
1. **감지 정확도 한계**: OpenCV 기반 접근법으로는 복잡한 UI에서 정확도가 떨어질 수 있음
2. **메신저 앱 다양성**: 카카오톡, LINE, WhatsApp 등 각기 다른 UI 패턴으로 인한 범용성 문제
3. **이미지 품질 의존성**: 저화질이나 압축된 스크린샷에서 감지 성능 저하 가능

### 비즈니스 리스크
1. **사용자 기대치 불일치**: "자동 크롭"에 대한 사용자의 기대가 실제 구현 능력을 초과할 수 있음
2. **확장성 제약**: 특정 도메인(메시지 화면)에 특화되어 범용성 부족

### 가장 취약한 가정
**메시지 화면에서 "캐릭터 이미지"가 시각적으로 일관되게 구분 가능하다는 가정**이 가장 취약하다. 이 가정이 틀릴 경우 핵심 자동 감지 기능이 실패하며, 수동 크롭 도구와 차별화되지 않는다.
