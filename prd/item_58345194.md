# 무추망망이 이미지 자동 크롭 시스템 PRD

## 개요

모바일 메신저 화면 스크린샷에서 이미지 영역을 자동으로 감지하고 추출하는 이미지 크롤링 시스템입니다. OpenCV를 활용하여 채팅 애플리케이션에서 공유된 이미지들을 자동으로 수집하고 처리하는 것이 핵심 목적입니다.

## 핵심 가치

**모바일 메신저 화면 스크린샷에서 이미지 부분만을 정확하게 자동 추출하여, 채팅 앱의 이미지를 수동 작업 없이 수집할 수 있게 하는 것**

이는 기존의 일반적인 이미지 크롭 도구와는 달리, 특정 모바일 UI 패턴(채팅 버블, 메시지 레이아웃)을 이해하고 그 안에서 이미지 컨텐츠만을 분리해내는 특화된 기능입니다.

## MVP 범위

### 포함되는 기능
1. **스크린샷 입력 처리**: 모바일 메신저 화면 스크린샷 파일을 입력으로 받기
2. **이미지 영역 자동 감지**: OpenCV를 사용하여 채팅 화면 내의 이미지 부분 감지
3. **자동 크롭 및 추출**: 감지된 이미지 영역을 정확히 크롭하여 별도 파일로 저장
4. **기본 출력 관리**: 추출된 이미지들을 지정된 폴더에 저장

### 명시적 제외사항 (v1에서 제외)
- 실시간 화면 캡처 기능
- 여러 메신저 앱 동시 지원 (단일 메신저 UI 패턴에 집중)
- 이미지 품질 향상/필터링 기능
- 웹 인터페이스 (커맨드라인 도구로 시작)
- 자동 분류/태깅 기능
- 클라우드 저장소 연동

## 요구사항

### 기능 요구사항
1. **입력 처리**
   - PNG, JPG 형식의 스크린샷 파일 지원
   - 단일 파일 및 폴더 단위 배치 처리 지원

2. **이미지 감지 로직**
   - 채팅 버블 내부의 이미지 컨텐츠 영역 식별
   - 텍스트, UI 요소와 구분하여 실제 이미지만 추출
   - 최소 크기 필터링 (너무 작은 아이콘/이모지 제외)

3. **크롭 및 저장**
   - 감지된 이미지의 경계를 정확히 크롭
   - 원본 해상도 유지
   - 순차적 파일명으로 저장 (예: extracted_001.jpg)

### 성능 요구사항
- 1080x1920 해상도 스크린샷 기준 5초 이내 처리
- 메모리 사용량 500MB 이하 유지

### 사용성 요구사항
- 커맨드라인 인터페이스로 간단한 실행
- 처리 진행상황 표시
- 성공/실패 결과 리포트

## 기술 고려사항

### 핵심 기술 스택
- **OpenCV**: 이미지 처리 및 컴퓨터 비전 작업
- **Python**: 주 개발 언어 (OpenCV Python 바인딩 활용)

### 이미지 감지 접근 방법 (가정)
- 색상 기반 영역 분할로 UI 요소와 컨텐츠 구분
- 윤곽선 검출을 통한 이미지 경계 식별
- 종횡비 및 크기 기반 필터링으로 실제 이미지만 추출

### 아키텍처 가정
```
입력 스크린샷 → 전처리 → 영역 감지 → 크롭 → 후처리 → 저장
```

## 위험요소 및 가정사항

### 핵심 가정사항
1. **단일 메신저 UI 패턴**: 초기 버전은 특정 메신저 앱의 UI 레이아웃에 최적화
2. **표준 해상도**: 일반적인 모바일 화면 해상도 범위 내에서 동작
3. **이미지 품질**: 스크린샷 품질이 분석에 충분한 수준

### 가장 취약한 가정
**채팅 앱의 UI 디자인이 일관되다는 가정** - 메신저 앱 업데이트로 인한 UI 변경, 다크모드/라이트모드 차이, 다양한 기기별 UI 변화가 감지 알고리즘을 무력화할 수 있음

### 기술적 위험
- OpenCV 기반 이미지 감지의 정확도 한계
- 복잡한 배경에서의 이미지 영역 오인식
- 다양한 이미지 형태(스티커, GIF, 동영상 썸네일)에 대한 처리 복잡성

### 완화 전략
- 프로토타입을 통한 감지 알고리즘 정확도 검증 우선 실시
- 테스트 데이터셋 구축하여 다양한 시나리오 검증
- 수동 보정 기능으로 자동화 한계 보완
