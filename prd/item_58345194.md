# 무추망망이 이미지 자동 크롭 시스템 PRD

## 개요

무추망망이 프로젝트의 일환으로 개발되는 이미지 자동 크롭 시스템. 스마트폰 메시지 앱 스크린샷에서 캐릭터 이미지를 자동으로 감지하고 추출하는 데스크톱 애플리케이션이다. OpenCV를 활용한 컴퓨터 비전 기술로 사용자의 수동 크롭 작업을 자동화한다.

**프로젝트 배경**: "무추망망이 집 만들기" 프로젝트에서 메시지 앱 스크린샷으로부터 캐릭터 이미지를 수집하고 정리하는 반복적인 작업의 자동화 필요성에서 출발.

## 핵심 가치

**스마트폰 메시지 스크린샷에서 캐릭터 이미지의 완전 자동 추출**

사용자가 메시지 앱 스크린샷을 입력하면, 수동 선택 없이 캐릭터 영역을 자동으로 감지하고 정확히 크롭된 개별 이미지로 저장한다. 기존의 수동 크롭 과정을 완전히 제거하여 시간을 절약하고 일관된 품질의 결과물을 생성한다.

## MVP 범위

### 포함 기능
- **자동 캐릭터 감지**: OpenCV를 사용한 색상 기반 경계 검출
- **정확한 크롭**: 감지된 캐릭터 영역의 최소 경계 상자 추출
- **배치 처리**: 단일 스크린샷에서 여러 캐릭터 동시 감지 및 추출
- **결과 저장**: 크롭된 이미지의 자동 파일명 생성 및 저장
- **기본 UI**: 이미지 로드, 처리 실행, 결과 확인이 가능한 간단한 인터페이스

### 명시적 제외사항 (v1에서)
- **다양한 앱 지원**: 메시지 앱 외의 다른 앱 스크린샷 처리
- **고급 이미지 편집**: 크롭 후 리사이징, 필터, 보정 기능
- **클라우드 통합**: 온라인 저장소 연동
- **실시간 처리**: 스크린샷 자동 감지 및 실시간 크롭
- **머신러닝 기반 객체 인식**: 딥러닝 모델을 사용한 복잡한 객체 감지

## 기능 요구사항

### 1. 이미지 입력 시스템
- 로컬 파일 선택 (jpg, png, bmp 지원)
- 드래그앤드롭 지원
- 입력 이미지 유효성 검사

### 2. 캐릭터 감지 엔진
- 색상 임계값 기반 객체 분리
- 윤곽선 검출 및 필터링 (최소 크기 조건)
- 노이즈 제거 (작은 객체 무시)
- 경계 상자 계산

### 3. 크롭 처리 시스템
- 감지된 각 영역의 최소 경계 사각형 생성
- 원본 화질 유지한 크롭 실행
- 순차적 파일명 생성 (예: character_1.png, character_2.png)

### 4. 사용자 인터페이스
- 원본 이미지 미리보기
- 감지된 영역 하이라이트 표시
- 크롭 실행 버튼
- 결과 이미지들의 썸네일 표시
- 저장 폴더 선택

## 기술 고려사항

### 아키텍처
- **언어**: Python (OpenCV 생태계 호환성)
- **GUI 프레임워크**: tkinter (Python 내장, 배포 편의성)
- **이미지 처리**: OpenCV 4.x + NumPy
- **파일 시스템**: pathlib (크로스 플랫폼 호환성)

### 핵심 알고리즘 (가정)
1. **전처리**: 색공간 변환 (BGR → HSV)
2. **색상 필터링**: HSV 범위 기반 마스크 생성
3. **모폴로지 연산**: 노이즈 제거 및 객체 정리
4. **윤곽선 검출**: cv2.findContours()
5. **필터링**: 면적 기준 유효한 윤곽선만 선택
6. **경계 상자**: cv2.boundingRect()로 크롭 영역 계산

### 성능 목표
- 1920x1080 스크린샷 처리 시간: 3초 이내
- 메모리 사용량: 500MB 이하
- 캐릭터 감지 정확도: 90% 이상 (일반적인 메시지 앱 스크린샷 기준)

## 위험요소 및 가정사항

### 핵심 가정사항
1. **색상 일관성**: 메시지 앱 캐릭터들이 배경과 구분 가능한 색상 대비를 가짐
2. **배경 단순성**: 메시지 앱 배경이 비교적 단색이거나 단순한 패턴
3. **캐릭터 크기**: 감지할 캐릭터가 최소 30x30 픽셀 이상의 크기
4. **스크린샷 품질**: 압축이나 블러로 인한 심각한 화질 저하 없음

### 기술적 위험요소
- **조명 변화**: 스크린 밝기나 색온도 변화로 인한 색상 감지 실패
- **복잡한 배경**: 캐릭터와 유사한 색상의 배경 요소로 인한 오감지
- **부분 가림**: 다른 UI 요소에 의해 캐릭터가 부분적으로 가려진 경우

### 가장 취약한 가정
**메시지 앱 캐릭터가 항상 배경과 명확한 색상 대비를 가진다**는 가정이 깨질 경우 전체 시스템의 감지 정확도가 크게 저하될 수 있음. 이 경우 색상 기반 접근법의 한계로 인해 더 복잡한 객체 감지 알고리즘이 필요할 수 있음.

### 완화 전략
- 다양한 색공간(HSV, LAB) 조합 실험
- 사용자가 색상 임계값을 수동 조정할 수 있는 옵션 제공
- 감지 결과 미리보기를 통한 사전 확인 기능
