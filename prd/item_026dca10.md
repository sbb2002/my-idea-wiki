# 주식 차트 인터랙티브 분석 인터페이스 PRD

## Overview

사용자가 CSV 주식 데이터를 업로드하여 인터랙티브한 차트를 생성하고, 두 점 클릭을 통한 구간별 상세 분석을 제공하는 웹 애플리케이션. 핵심 차별화 요소는 P1-P2 두 점 선택을 통한 즉석 수익률 분석과 성능 최적화를 위한 동적 일봉/월봉 전환 시스템이다.

## Core Value

**CSV 주식 데이터를 업로드하여 두 점(P1, P2) 클릭만으로 해당 구간의 수익률, 최고/최저점 등 상세 투자 성과 분석을 즉시 확인할 수 있는 인터랙티브 차트 도구**

## MVP Scope

### 포함 기능

**필수 기능:**
- CSV 파일 업로드 (날짜, 시가, 고가, 저가, 종가 컬럼 필수)
- 선형 차트 (종가 기준) 및 캔들 차트 전환
- 두 점(P1, P2) 클릭을 통한 구간 선택
- 선택 구간에 대한 분석 테이블:
  - 시작/끝 시간 및 기간 길이
  - 시작점/끝점 가격
  - 수익률 = (끝점 가격 - 시작점 가격) / 시작점 가격
  - 수익률 속도 = 수익률 / 기간길이
  - 기간 내 최저점/최고점 가격 및 시각
  - 최저점→최고점 시간 차이
- 상단 제목 박스 (페이지 이름 편집 가능)
- JSON 형태 저장/불러오기

**성능 최적화:**
- 확대 수준에 따른 동적 일봉/월봉 전환
- 드래그를 통한 확대/축소, 구간 이동

**추가 차트 기능:**
- Y축 로그 변환
- 축 고정 기능

### 명시적 제외 사항

- 실시간 데이터 연동 (CSV 업로드만 지원)
- 복수 종목 동시 비교
- 기술적 지표 (이동평균, RSI 등)
- 사용자 계정 관리 또는 클라우드 저장
- 모바일 최적화 (데스크톱 우선)
- 알림 또는 자동화 기능

## Requirements

### 데이터 요구사항

**CSV 입력 형식:**
- 필수 컬럼: 날짜(Date), 시가(Open), 고가(High), 저가(Low), 종가(Close)
- 날짜 형식: YYYY-MM-DD 또는 자동 파싱 가능한 표준 형식
- 숫자 데이터: 소수점 포함 가능한 숫자 형식
- 정렬: 날짜순 오름차순 (자동 정렬 기능 포함)

### UI/UX 요구사항

**레이아웃:**
- 상단: 편집 가능한 제목 박스
- 중앙: 주 차트 영역 (화면 높이의 60% 이상)
- 하단: 구간 분석 테이블 (P1, P2 선택 시에만 표시)
- 좌측 또는 상단: 컨트롤 패널 (차트 타입, 축 옵션 등)

**인터랙션:**
- P1, P2 점 클릭: 마우스 클릭으로 두 점 선택, 시각적 마커 표시
- 드래그 확대/축소: 마우스 드래그로 특정 영역 확대
- 팬 기능: 드래그로 차트 이동
- 차트 타입 전환: 버튼 클릭으로 선형/캔들 전환

### 분석 계산 요구사항

**수익률 계산:**
```
수익률 = (P2_가격 - P1_가격) / P1_가격 * 100
수익률_속도 = 수익률 / 기간_길이(일)
```

**기간 내 최고/최저점:**
- P1과 P2 사이 모든 데이터 포인트에서 최고가/최저가 검색
- 해당 가격의 정확한 날짜와 시각 기록

### 성능 요구사항

**동적 렌더링:**
- 표시 기간이 365일 초과 시 월봉 데이터로 자동 집계
- 표시 기간이 365일 이하 시 일봉 데이터 표시
- 줌 레벨 변경 시 실시간 전환 (라그 최대 500ms)

**데이터 처리:**
- 최대 10,000개 데이터 포인트까지 지원
- CSV 파일 크기 제한: 5MB

## Technical Considerations

### 프론트엔드 기술 스택

**차트 라이브러리:**
- Chart.js 또는 D3.js 기반 구현 권장
- 캔들스틱 차트 지원 필수
- 실시간 인터랙션 지원

**파일 처리:**
- JavaScript FileReader API로 CSV 파싱
- PapaParse 라이브러리 활용 권장

### 데이터 구조

**내부 데이터 모델:**
```javascript
{
  metadata: {
    title: string,
    uploadDate: timestamp,
    dataPoints: number
  },
  rawData: [
    {
      date: Date,
      open: number,
      high: number,
      low: number,
      close: number
    }
  ],
  monthlyData: [...], // 동적 생성
  analysis: {
    p1: {point, date, price},
    p2: {point, date, price},
    metrics: {...}
  }
}
```

### 저장 형식

**JSON 출력:**
- 전체 차트 상태 (원본 데이터 + 분석 결과 + UI 설정)
- 브라우저 로컬 저장 또는 파일 다운로드
- 동일한 JSON으로 완전한 상태 복원 가능

## Risks & Assumptions

### 핵심 가정

**가장 약한 가정:** 사용자가 표준화된 CSV 형식으로 데이터를 제공한다는 가정
- **리스크:** 다양한 CSV 형식, 컬럼명, 날짜 형식으로 인한 파싱 실패
- **완화책:** 유연한 CSV 파싱 로직 및 사용자 컬럼 매핑 UI 필요

### 기술적 제약사항

**브라우저 성능:**
- 대용량 데이터셋(10,000+ 포인트) 처리 시 렌더링 지연
- **완화책:** 가상화 스크롤링 또는 데이터 샘플링 구현

**메모리 제약:**
- 일봉/월봉 데이터 동시 보관 시 메모리 사용량 증가
- **완화책:** 월봉 데이터 온디맨드 생성 및 캐싱

### 사용성 리스크

**P1, P2 선택 UX:**
- 정확한 점 클릭이 어려울 수 있음 (특히 밀집된 데이터)
- **완화책:** 클릭 허용 반경 확대 및 스냅-투-포인트 기능

**차트 반응성:**
- 실시간 드래그/줌 시 성능 저하 가능성
- **완화책:** 디바운싱 및 프레임 레이트 제한 적용

### 데이터 품질 리스크

**잘못된 데이터:**
- 누락된 날짜, 비정상적인 가격 데이터
- **완화책:** 데이터 유효성 검증 및 오류 메시지 표시

**시간대 처리:**
- CSV의 날짜가 다른 시간대를 나타낼 수 있음
- **가정:** 모든 날짜를 로컬 시간대로 처리
