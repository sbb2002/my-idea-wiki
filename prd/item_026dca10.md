# 주식 차트 인터랙티브 분석 인터페이스 PRD

## 개요

CSV 형태의 주식 데이터를 업로드하여 인터랙티브한 차트 분석을 수행하는 웹 애플리케이션입니다. 사용자는 드래그를 통한 구간 선택, 두 점 클릭을 통한 구간별 분석, 다양한 차트 옵션 설정 등을 통해 주식 데이터를 심층적으로 분석할 수 있습니다. 성능 최적화를 위한 스마트 렌더링과 데이터 저장/불러오기 기능을 포함합니다.

## 목표

### 주요 목표
- CSV 데이터를 통한 직관적인 주식 차트 시각화 제공
- 드래그 기반 구간 선택과 두 점 클릭을 통한 상세 분석 기능 구현
- 대용량 데이터 처리를 위한 성능 최적화된 렌더링 시스템 구축
- 분석 결과의 저장 및 공유 기능 제공

### 사용자 경험 목표
- 금융 데이터 분석에 익숙하지 않은 사용자도 쉽게 사용할 수 있는 직관적 UI
- 빠른 응답성과 원활한 인터랙션 경험
- 상세한 분석 지표를 통한 의미있는 인사이트 제공

## 기능 요구사항

### 1. 데이터 입력 및 관리

#### 1.1 CSV 데이터 업로드
- **필수 컬럼**: Date (YYYY-MM-DD 형식), Close (종가)
- **선택 컬럼**: Open (시가), High (고가), Low (저가), Volume (거래량)
- **지원 형식**: CSV 파일 (UTF-8 인코딩)
- **최대 파일 크기**: 10MB
- **데이터 검증**: 
  - 날짜 형식 유효성 검사
  - 숫자 데이터 유효성 검사
  - 중복 날짜 데이터 처리
  - 누락된 데이터 포인트 식별 및 알림

#### 1.2 프로젝트 관리
- **제목 설정**: 상단에 편집 가능한 제목 박스 배치
- **저장/불러오기**: JSON 형태로 프로젝트 데이터 저장 및 불러오기
- **JSON 구조**:
  ```json
  {
    "title": "프로젝트 제목",
    "data": [...], 
    "settings": {...},
    "analysis": [...]
  }
  ```

### 2. 차트 시각화

#### 2.1 기본 차트 표시
- **차트 타입**: 선형 차트 (기본), 캔들스틱 차트 (옵션)
- **기본 색상**: 황금색 (#FFD700) 선
- **축 설정**:
  - X축: 날짜/시간
  - Y축: 가격 (원, 달러 등)
  - Y축 로그 변환 옵션

#### 2.2 스마트 렌더링 시스템
- **렌더링 로직**:
  - 표시 기간 > 2년: 월봉 차트
  - 표시 기간 6개월~2년: 주봉 차트  
  - 표시 기간 < 6개월: 일봉 차트
- **성능 최적화**: 화면에 표시되는 데이터 포인트만 렌더링
- **부드러운 전환**: 확대/축소 시 차트 타입 자동 전환

#### 2.3 차트 컨트롤
- **확대/축소**: 마우스 휠, 핀치 제스처
- **패닝**: 드래그를 통한 좌우 이동
- **축 고정**: X축, Y축 각각 고정/해제 옵션
- **리셋**: 원본 뷰로 되돌리기 버튼

### 3. 인터랙티브 분석 기능

#### 3.1 드래그 구간 선택
- **선택 방법**: 마우스 드래그로 시간 범위 선택
- **시각적 피드백**: 선택 영역 하이라이트 표시
- **자동 확대**: 선택된 구간으로 차트 자동 확대

#### 3.2 두 점 클릭 분석
- **포인트 선택**: P1, P2 순서로 차트 상의 두 점 클릭
- **시각적 표시**: 
  - P1: 파란색 원형 마커
  - P2: 파란색 원형 마커
  - 두 점을 잇는 파란색 직선
- **마커 정보**: 각 포인트에 날짜와 가격 툴팁 표시

### 4. 분석 데이터 테이블

#### 4.1 기본 분석 지표
선택된 구간에 대한 다음 정보를 하단 테이블에 표시:

| 항목 | 내용 | 계산 방법 |
|------|------|-----------|
| 시작 시간 | P1 날짜 | 사용자 클릭 포인트 |
| 끝 시간 | P2 날짜 | 사용자 클릭 포인트 |
| 기간 길이 | 일 수 | P2 날짜 - P1 날짜 |
| 시작점 가격 | P1 가격 | 사용자 클릭 포인트 |
| 끝점 가격 | P2 가격 | 사용자 클릭 포인트 |
| 수익률 | 백분율 | ((P2가격 - P1가격) / P1가격) × 100 |
| 수익률속도 | 연율화 | (수익률 / 기간일수) × 365 |

#### 4.2 고급 분석 지표
| 항목 | 내용 | 계산 방법 |
|------|------|-----------|
| 구간 최고가 | 기간 내 최고 가격 | MAX(구간 내 모든 가격) |
| 구간 최저가 | 기간 내 최저 가격 | MIN(구간 내 모든 가격) |
| 최고가 달성일 | 최고가 날짜 | 최고가에 해당하는 날짜 |
| 최저가 달성일 | 최저가 날짜 | 최저가에 해당하는 날짜 |
| 변동성 | 표준편차 | 구간 내 일별 수익률의 표준편차 |
| 최대 낙폭 | 백분율 | 구간 내 최대 하락폭 |

- **로딩 상태**: 데이터 처리 중 로딩 인디케이터 표시

### 6. 데이터 저장 및 공유

#### 6.1 프로젝트 저장
- **저장 형식**: JSON
- **저장 내용**: 
  - 원본 CSV 데이터
  - 현재 차트 설정
  - 선택된 분석 포인트
  - 사용자 설정 제목
- **저장 위치**: 브라우저 로컬 스토리지 + 다운로드 옵션

#### 6.2 내보내기 기능
- **차트 이미지**: PNG, SVG 형태로 내보내기
- **분석 데이터**: CSV, Excel 형태로 내보내기
- **종합 보고서**: PDF 형태로 차트와 분석 데이터 통합 내보내기

## 기술적 고려사항

### 1. 프론트엔드 기술 스택

#### 1.1 필수 기술
- **기본 프레임워크**: React 또는 Vue.js
- **차트 라이브러리**: D3.js 또는 Chart.js
- **상태 관리**: Redux/Vuex (대용량 데이터 처리용)
- **파일 처리**: Papa Parse (CSV 파싱)
- **UI 컴포넌트**: 커스텀 컴포넌트 또는 Ant Design/Vuetify

#### 1.2 성능 최적화
- **가상화**: 큰 데이터셋을 위한 윈도윙/가상화 구현
- **메모이제이션**: React.memo 또는 computed 속성 활용
- **지연 로딩**: 필요에 따른 데이터 청크 로딩
- **웹 워커**: 대용량 데이터 처리를 위한 백그라운드 처리

### 2. 데이터 처리 아키텍처

#### 2.1 데이터 파이프라인
```
CSV 업로드 → 검증 → 파싱 → 정제 → 인덱싱 → 캐싱 → 렌더링
```

#### 2.2 데이터 구조
```javascript
// 기본 데이터 포인트
interface DataPoint {
  date: Date;
  open?: number;
  high?: number;
  low?: number;
  close: number;
  volume?: number;
}

// 분석 결과
interface AnalysisResult {
  p1: DataPoint;
  p2: DataPoint;
  metrics: {
    duration: number;
    return: number;
    annualizedReturn: number;
    maxPrice: number;
    minPrice: number;
    volatility: number;
    maxDrawdown: number;
  };
}
```

### 3. 렌더링 최적화

#### 3.1 LOD (Level of Detail) 시스템
- **확대 수준별 데이터 밀도 조정**
- **동적 데이터 샘플링**
- **캐시된 렌더링 결과 재사용**

#### 3.2 Canvas vs SVG 선택 기준
- **대용량 데이터 (>10,000 포인트)**: Canvas 사용
- **중간 데이터 (1,000-10,000 포인트)**: SVG 사용
- **소량 데이터 (<1,000 포인트)**: SVG 사용

### 4. 브라우저 호환성

#### 4.1 지원 브라우저
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

#### 4.2 필수 Web API
- File API (파일 업로드)
- Canvas API (차트 렌더링)
- Local Storage API (데이터 저장)
- Blob API (파일 다운로드)

### 5. 보안 고려사항

#### 5.1 데이터 보안
- **클라이언트 사이드 처리**: 모든 데이터는 브라우저 내에서만 처리
- **민감 정보 제거**: 업로드된 파일에서 개인정보 자동 식별 및 제거 옵션
- **HTTPS 필수**: 모든 통신은 HTTPS를 통해서만 수행

#### 5.2 파일 검증
- **파일 크기 제한**: 10MB 이하로 제한
- **파일 형식 검증**: CSV 확장자 및 MIME 타입 검사
- **악성 코드 검사**: 업로드된 파일 내용 기본 검증
