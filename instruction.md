# 아이디어 위키 — 사용 설명서

> 서버는 이미 Render에 배포된 상태를 가정합니다.

---

## 1. 최초 설정

### 1-1. 텔레그램 봇 만들기

1. 텔레그램에서 **@BotFather** 검색 → `/newbot` 명령 전송
2. 봇 이름과 username 입력 (username은 `_bot`으로 끝나야 함)
3. BotFather가 발급한 **봇 토큰** 복사 (예: `123456789:ABCdef...`)

**Chat ID 확인 방법:**

1. 방금 만든 봇에게 아무 메시지나 전송
2. 브라우저에서 아래 URL 접속 (토큰 교체)
   ```
   https://api.telegram.org/bot<봇토큰>/getUpdates
   ```
3. 응답 JSON에서 `"chat": {"id": 123456789}` 값이 Chat ID

---

### 1-2. Google Drive 폴더 설정

**폴더 구조:**
```
내 드라이브/
└── my-idea-wiki/
    ├── notes/      ← 여기에 아이디어 노트를 업로드
    └── wikis/      ← 위키 결과물이 저장됨 (건드리지 않아도 됨)
```

**폴더 ID 확인:**
구글 드라이브에서 폴더를 열었을 때 URL의 마지막 부분이 폴더 ID입니다.
```
https://drive.google.com/drive/folders/1ABC123DEF456
                                        ↑ 이 부분
```

**서비스 계정 공유 설정 (필수):**
`notes`와 `wikis` 폴더 모두에 대해:
1. 폴더 우클릭 → **공유**
2. 서비스 계정 이메일 입력 (개발자에게 문의)
3. 권한: **편집자** 선택 후 공유

**⚠️ wikis 폴더에 빈 파일 미리 업로드 (필수):**

서비스 계정은 구글 드라이브에 파일을 새로 생성할 수 없어요.
`wikis` 폴더에 아래 두 파일을 **직접 미리 업로드**해야 합니다.

> **업로드 전 필수 설정:** 구글 드라이브 웹 → 우측 상단 톱니바퀴(⚙️) → 설정 →
> **"업로드된 파일을 Google 문서 편집기 형식으로 변환" 체크 해제**

업로드할 파일:
- `wiki.json` — 아무 텍스트 에디터로 생성, 내용은 `{}` 또는 비워도 됨
- `index.html` — 개발자에게 받거나 레포의 `viewer/index.html` 사용

---

### 1-3. Render 환경변수 설정

개발자에게 아래 값들을 전달해서 Render 환경변수에 등록하면 설정 완료입니다:

| 항목 | 값 |
|------|-----|
| 텔레그램 봇 토큰 | BotFather에서 발급받은 토큰 |
| 텔레그램 Chat ID | 위에서 확인한 Chat ID |
| notes 폴더 ID | 구글 드라이브 notes 폴더 ID |
| wikis 폴더 ID | 구글 드라이브 wikis 폴더 ID |

## 2. 노트 작성법

### 지원 파일 형식
- 일반 텍스트 (`.txt`)
- 구글 독스 (드라이브에서 직접 작성)

### 태그 사용
노트 본문 어디에나 `#태그명` 형식으로 태그를 달 수 있습니다.
태그가 있으면 AI 자동 분류보다 우선 적용됩니다.

```
오늘 RPG 전투 시스템에 대한 아이디어를 정리했다.

적의 약점을 분석해서 속성 공격을 하면 추가 대미지를 주는 방식으로...

#게임 #RPG #전투로직
```

### 기존 아이디어 추가/보완
이전에 위키화된 아이디어에 새 내용을 추가하고 싶으면,
**같은 주제로 새 노트를 작성해서 업로드**하면 됩니다.
AI가 기존 아이템을 인식하고 버전을 누적합니다.

## 3. 텔레그램 봇 사용법

봇에게 아래 명령어를 전송하세요.

| 명령어 | 동작 |
|--------|------|
| `/run` | 즉시 위키화 실행 |
| `/status` | 마지막 실행 결과 조회 |
| `/schedule` | 현재 자동 실행 주기 확인 |
| `/set <주기>` | 자동 실행 주기 변경 |
| `/cancel` | 예약된 실행 취소 |

**주기 설정 예시:**
```
/set 매주 월요일 오전 9시
/set 매일 자정
/set 0 9 * * 1   (cron 표현식 직접 입력도 가능)
```

**알람 종류:**

| 아이콘 | 의미 |
|--------|------|
| ✅ | 위키화 완료 — 처리/신규/업데이트 수 요약 |
| ⚠️ | 일부 실패 — 성공한 부분 결과와 함께 안내 |
| ❌ | 전체 실패 — Claude, Gemini 모두 실패 |
| ⏳ | 서버 슬립 중 — `/run` 후 콜드 스타트 대기 안내 |

> **콜드 스타트 안내:** 텔레그램 Webhook 서버는 15분 이상 미사용 시 슬립 상태가 됩니다.
> 슬립 중 `/run`을 보내면 "잠시만 기다려주세요" 메시지가 오고 30초~1분 후 작업이 시작됩니다.

## 4. 결과 확인 — 위키 뷰어 열기

위키 결과물은 `wikis` 폴더의 `index.html`로 확인합니다.

### 방법 A: 다운로드해서 열기 (간단)
1. 구글 드라이브에서 `wikis` 폴더 접속
2. `index.html` 우클릭 → **다운로드**

   > ⚠️ 드라이브 웹에서 직접 클릭하면 구글 독스로 열림 — 반드시 **다운로드** 후 열 것

3. 다운로드된 파일을 더블클릭 → 브라우저에서 열림

### 방법 B: 구글 드라이브 데스크탑 앱으로 자동 동기화 (권장)
1. [구글 드라이브 데스크탑 앱](https://www.google.com/drive/download/) 설치
2. 앱 설정에서 `wikis` 폴더를 로컬 동기화 대상에 추가
3. 로컬 탐색기에서 `index.html` 더블클릭

위키화가 완료될 때마다 자동 동기화되어 항상 최신 결과를 볼 수 있습니다.

## 5. 위키 뷰어 UI 안내

### 화면 구성

```
┌─────────────────────────────────────────────────────┐
│ 💡 아이디어 위키  [검색창]  [위키|그래프]  [🌙다크]  │  ← 헤더
├──────────────┬──────────────────────────┬───────────┤
│              │  아이템 제목             │  목차     │
│  아이템 목록  │  ─────────────────────  │  · 개요   │
│              │  [인포박스]  개요 텍스트  │  · 히스토리│
│  · 아이템 A  │                          │  · 연관   │
│  · 아이템 B  │  버전 히스토리           │           │
│  · 아이템 C  │  연관 아이템             │           │
└──────────────┴──────────────────────────┴───────────┘
```

### 주요 기능

**아이템 목록 (왼쪽 사이드바)**
- 전체 아이템이 목록으로 표시
- 클릭하면 해당 위키 문서로 이동

**검색**
- 헤더의 검색창에 입력하면 제목·태그·요약에서 실시간 필터링

**위키 문서 본문**
- **인포박스**: 우측 상단의 요약 박스 (태그, 버전 수, 최신 날짜, 연관 아이템)
- **개요**: AI가 작성한 아이디어 현재 상태 요약
- **버전 히스토리**: 타임라인 형식으로 아이디어 발전 과정 표시
- **연관 아이템**: 클릭하면 해당 아이템으로 바로 이동

**그래프 뷰**
- 헤더의 **[그래프]** 버튼으로 전환
- 아이템 간 연관관계를 노드-엣지 그래프로 시각화
- 노드 클릭 → 해당 위키 문서로 이동
- 드래그로 노드 이동, 스크롤로 줌 인/아웃
- 우측 하단 `+` `-` `⌂` 버튼으로 줌 조절 및 초기화

**테마 전환**
- 헤더 우측 **[🌙 다크]** / **[☀️ 라이트]** 버튼으로 테마 전환
- 선택한 테마는 다음에 열어도 유지됨
